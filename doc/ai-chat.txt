1. INTRODUCTION                                         *ai-chat-introduction*

ai-chat.nvim is a transparent, native-feeling AI chat interface for Neovim.
It provides a vertical split chat panel where you can have multi-turn
conversations with AI models while keeping your code visible alongside.

Design principles:
- Transparency: always see what context the AI receives and what it costs
- Native: uses splits, buffers, and standard neovim keybindings
- Minimal: does one thing well — chat with an AI about your code
- Local-first: defaults to Ollama (free, private, zero-config)

==============================================================================
2. REQUIREMENTS                                         *ai-chat-requirements*

- Neovim >= 0.10
- curl (for HTTP requests)
- An AI provider:
  - Ollama (recommended, https://ollama.ai) — local, free, private
  - Anthropic API key (for Claude models)
  - OpenAI API key (or compatible endpoint)

Run `:checkhealth ai-chat` to verify your setup (neovim version, curl,
provider reachability, treesitter parser, storage directories).

==============================================================================
3. SETUP                                                       *ai-chat-setup*

Add to your plugin manager (e.g., lazy.nvim): >lua

    {
        "your-username/ai-chat.nvim",
        cmd = { "AiChat", "AiChatOpen", "AiChatSend" },
        keys = { "<leader>aa", "<leader>ac" },
        config = function()
            require("ai-chat").setup({
                default_provider = "ollama",
                default_model = "llama3.2",
            })
        end,
    }
<

Minimal setup (uses all defaults): >lua

    require("ai-chat").setup()
<

4. COMMANDS                                                 *ai-chat-commands*

:AiChat                                                            *:AiChat*
    Toggle the chat panel open/closed.

:AiChatOpen                                                    *:AiChatOpen*
    Open the chat panel (no-op if already open).

:AiChatClose                                                  *:AiChatClose*
    Close the chat panel.

:AiChatSend [text]                                             *:AiChatSend*
    Send a message. If [text] is omitted, reads from the input area.

:AiChatClear                                                  *:AiChatClear*
    Clear the current conversation and start fresh.

:AiChatModel [model]                                          *:AiChatModel*
    Switch the active model. Opens a picker if [model] is omitted.

:AiChatProvider [provider]                                *:AiChatProvider*
    Switch the active provider. Opens a picker if [provider] is omitted.

:AiChatHistory                                            *:AiChatHistory*
    Browse saved conversation history.

:AiChatSave [name]                                            *:AiChatSave*
    Save the current conversation with an optional name.

:AiChatLog                                                    *:AiChatLog*
    Open the audit log showing API interactions.

:AiChatCosts                                                  *:AiChatCosts*
    Show a summary of API costs for this session.

:AiChatKeys                                                    *:AiChatKeys*
    Show all active keybindings.

:AiChatConfig                                                *:AiChatConfig*
    Show the resolved plugin configuration.

5. KEYBINDINGS                                           *ai-chat-keybindings*

Global (default prefix: <leader>a):~

    <leader>aa     Toggle chat panel
    <leader>as     Send visual selection to chat (visual mode)
    <leader>ae     Explain visual selection (visual mode)
    <leader>af     Fix visual selection (visual mode)
    <leader>ac     Focus chat input
    <leader>am     Switch model
    <leader>ap     Switch provider
    <leader>ar     Open proposal quickfix list
    <leader>an     Jump to next pending proposal (across files)
    <leader>aR     Accept all pending proposals (with confirmation)
    <leader>ag     Quick annotate (pre-fill /annotate @buffer)
    <leader>aA     Expand/collapse all annotations in buffer
    <leader>ax     Clear all annotations from buffer

Chat buffer:~

    q              Close chat panel
    <C-c>          Cancel active generation
    ]]             Jump to next message
    [[             Jump to previous message
    ]c             Jump to next code block (chat only; ]c in diff = next hunk)
    [c             Jump to previous code block (chat only; [c in diff = prev hunk)
    gY             Yank code block under cursor
    ga             Apply code block via diff
    gO             Open code block in a new split
    i              Focus input area
    ?              Show keybinding help

Input buffer:~

    <CR>           Send message (normal mode)
    <C-CR>         Send message (insert mode)
    <CR>           Insert newline (insert mode)
    <S-CR>         Insert newline (normal mode)
    <Up>           Recall previous message
    <Down>         Recall next message
    q              Close panel (when input is empty)
    <C-c>          Cancel active generation

Code buffer (active when proposals are pending):~

    gp             Review proposal at cursor (opens diff)
    ga             Review proposal at cursor (opens diff)
    gx             Reject/dismiss proposal at cursor

Code buffer (active when annotations are present):~

    ]a             Jump to next annotation
    [a             Jump to previous annotation
    za             Toggle expand/collapse annotation at cursor
    gx             Dismiss annotation at cursor

All keybindings can be overridden in setup(). Set any key to `false`
to disable it.

==============================================================================
6. SLASH COMMANDS                                     *ai-chat-slash-commands*

Type these in the input area:

    /clear          Clear conversation, start fresh
    /new            Save current and start new conversation
    /model [name]   Switch model (picker if no name given)
    /provider [n]   Switch provider (picker if no name given)
    /context        Show available context types
    /save [name]    Save conversation with optional name
    /load           Browse saved conversations
    /help           List available commands

Extended (v0.5+):~

    /annotate [@ctx] [prompt]  Place inline annotations on buffer/selection
    /annotate clear            Remove annotations from current buffer
    /annotate clear all        Remove annotations from all buffers

==============================================================================
7. CONTEXT REFERENCES                                      *ai-chat-context*

Add context to your messages by prefixing with @tags:

    @buffer         Include the current code buffer content
    @selection      Include the last visual selection
    @diagnostics    Include LSP diagnostics for current buffer
    @diff           Include git diff (unstaged changes)
    @file:path      Include a specific file

Example: >
    @buffer How do I fix the type error on line 42?
    @selection Explain this function
    @diff Review my changes
<

Context is displayed in message metadata so you always know what the AI sees.

==============================================================================
8. PROVIDERS                                              *ai-chat-providers*

Ollama (default):~
    Local inference. No API key needed. Install from https://ollama.ai
    Start with: `ollama serve`
    Pull a model: `ollama pull llama3.2`

Anthropic:~
    Set ANTHROPIC_API_KEY environment variable.

OpenAI-compatible:~
    Set OPENAI_API_KEY environment variable.
    Covers: OpenAI, Azure OpenAI, Groq, Together, LM Studio.

Amazon Bedrock:~
    Requires AWS CLI configured with appropriate credentials.

9. CONFIGURATION                                      *ai-chat-configuration*

Full configuration with defaults: >lua

    require("ai-chat").setup({
        default_provider = "ollama",
        default_model = "llama3.2",

        providers = {
            ollama = {
                host = "http://localhost:11434",
            },
            anthropic = {
                model = "claude-sonnet-4-20250514",
                max_tokens = 8192,
            },
            openai_compat = {
                endpoint = "https://api.openai.com/v1/chat/completions",
                model = "gpt-4o",
            },
        },

        ui = {
            width = 0.25,          -- fraction of editor width
            min_width = 60,
            max_width = 120,
            position = "right",    -- "right" or "left"
            input_height = 3,
            show_winbar = true,
            show_cost = true,
            show_tokens = true,
            spinner = true,
        },

        chat = {
            system_prompt = nil,   -- custom system prompt
            temperature = 0.7,
            max_tokens = 4096,
            context_window = nil,  -- max tokens for context (nil = provider default)
            thinking = false,
            auto_scroll = true,
            show_context = true,
        },

        history = {
            enabled = true,
            max_conversations = 100,
            storage_path = nil,    -- defaults to stdpath("data")/ai-chat
        },

        keys = {
            toggle = "<leader>aa",
            send_selection = "<leader>as",
            quick_explain = "<leader>ae",
            quick_fix = "<leader>af",
            focus_input = "<leader>ac",
            switch_model = "<leader>am",
            switch_provider = "<leader>ap",
            proposal_list = "<leader>ar",
            proposal_next = "<leader>an",
            proposal_accept_all = "<leader>aR",
            proposal_preview = "gp",
            proposal_accept = "ga",
            proposal_reject = "gx",
            annotate_quick = "<leader>ag",
            annotate_toggle_all = "<leader>aA",
            annotate_clear = "<leader>ax",
            annotate_next = "]a",
            annotate_prev = "[a",
            annotate_toggle = "za",
            annotate_dismiss = "gx",
        },

        proposals = {
            enabled = true,        -- enable proposal queue
            signs = true,          -- show signs in gutter
            virtual_text = true,   -- show virtual text hints
            quickfix = true,       -- populate quickfix list
            notify = true,         -- vim.notify on new proposals
        },

        annotations = {
            enabled = true,        -- enable inline annotations
            signs = true,          -- show signs in gutter
            virtual_text = true,   -- show collapsed summary text
            collapsed = true,      -- start collapsed (vs expanded)
            notify = true,         -- vim.notify on new annotations
        },

        log = {
            enabled = true,
            level = "info",
            max_size_mb = 10,
        },
    })
<

10. HIGHLIGHTS                                           *ai-chat-highlights*

ai-chat.nvim defines the following highlight groups (with defaults):

    AiChatUser            Title           Message header for user
    AiChatAssistant       Statement       Message header for assistant
    AiChatMeta            Comment         Metadata (tokens, cost, context)
    AiChatError           DiagnosticError Error messages
    AiChatWarning         DiagnosticWarn  Warning messages
    AiChatSpinner         DiagnosticInfo  Spinner animation
    AiChatSeparator       WinSeparator    Separator lines
    AiChatInputPrompt     Question        Input area prompt
    AiChatContextTag      Tag             Context reference tags
    AiChatProposalSign    DiagnosticSignHint  Pending proposal sign
    AiChatProposalExpired DiagnosticSignWarn  Expired proposal sign
    AiChatAnnotationSign  DiagnosticSignInfo  Annotation sign
    AiChatAnnotationText  Comment             Annotation summary (virt_text)
    AiChatAnnotationDetail Comment            Annotation detail (virt_lines)

Override in your colorscheme or config: >lua
    vim.api.nvim_set_hl(0, "AiChatUser", { fg = "#e0af68", bold = true })
<

11. EVENTS                                                  *ai-chat-events*

ai-chat.nvim emits User autocommand events:

    AiChatMessageSent        User sends a message
    AiChatResponseStart      Response streaming begins
    AiChatResponseDone       Response streaming completes
    AiChatResponseError      Response fails
    AiChatPanelOpened        Chat panel opens
    AiChatPanelClosed        Chat panel closes
    AiChatProviderChanged    Provider or model switched
    AiChatConversationCleared Conversation cleared
    AiChatProposalCreated    AI proposes a code change
    AiChatProposalAccepted   User accepts a proposal
    AiChatProposalRejected   User rejects a proposal
    AiChatProposalExpired    Proposal invalidated by user edits
    AiChatAnnotationCreated  Annotations placed on a buffer
    AiChatAnnotationCleared  Annotations removed from a buffer

Example: >lua
    vim.api.nvim_create_autocmd("User", {
        pattern = "AiChatResponseDone",
        callback = function(args)
            print("Response complete!")
        end,
    })
<
